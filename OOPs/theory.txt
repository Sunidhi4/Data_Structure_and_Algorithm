Constructor Chaining : **Constructor chaining** is a concept in object-oriented programming (OOP) where 
one constructor calls another constructor within the same class or in the superclass. This is typically
 done to reduce code duplication and ensure that objects are initialized correctly, especially when a 
 class has multiple constructors with different parameters.

### Types of Constructor Chaining:
1. **Within the same class (using `this`)**:
   In this case, one constructor calls another constructor within the same class. 
   This helps when multiple constructors are needed to handle different initialization scenarios 
   but share some common setup code.

   ```java
   class MyClass {
       int x;
       int y;

       // Constructor 1
       MyClass(int x) {
           this(x, 0);  // Calling Constructor 2
       }

       // Constructor 2
       MyClass(int x, int y) {
           this.x = x;
           this.y = y;
       }
   }
   ```

   In this example, when the first constructor is called with one argument (`MyClass(int x)`), 
   it invokes the second constructor (`MyClass(int x, int y)`) with a default value for `y`.

2. **Between subclass and superclass (using `super`)**:
   In Java and many other OOP languages, you can also chain constructors from a subclass to a 
   superclass using the `super()` keyword. This is often done to ensure that the parent class is properly
    initialized before initializing the subclass.

   ```java
   class Animal {
       String name;

       // Constructor in superclass
       Animal(String name) {
           this.name = name;
       }
   }

   class Dog extends Animal {
       int age;

       // Constructor in subclass
       Dog(String name, int age) {
           super(name);  // Calling superclass constructor
           this.age = age;
       }
   }
   ```

   In this example, the `Dog` constructor calls the `Animal` constructor using `super(name)` to initialize the
    `name` field in the parent class before initializing the `age` field in the `Dog` class.

### Benefits of Constructor Chaining:
- **Code Reusability**: By reusing constructors, you avoid repeating initialization code in each constructor.

- **Maintainability**: If initialization logic changes, you only need to modify one constructor instead 
of multiple ones.

- **Clearer Logic**: Constructor chaining clarifies the intention behind different constructor variations, 
making the code easier to follow and maintain.

### Rules in Java (and similar languages):
- Constructor chaining can happen **within the same class** using `this()` or **between parent and child classes**
 using `super()`.

- The constructor chaining must happen as the first statement in the constructor, whether using
 `this()` or `super()`.

In summary, constructor chaining is a technique used to call one constructor from another within the same 
class or between a subclass and superclass, making object initialization more efficient and manageable.
